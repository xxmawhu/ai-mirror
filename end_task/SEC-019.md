## Description

`cmd_rm()` derives `ai_user` from `project_path` but never calls `validate_ai_user_ownership()` to verify that the caller actually owns that ai-user. Any member of the `ai-mirror` group can delete any other user's ai-user by passing their project path.

## Attack Scenario

1. User A (ai-mirror group member) runs `sudo am rm /home/userB/ai-user-project`
2. `cmd_rm` derives `ai_user = "ai-userB"` from the path
3. No ownership check — user A is not userB
4. The ai-user's home, SSH keys, and bind mounts are destroyed

## Affected Code

```cpp
// src/cli/commands.cpp:479-537
int cmd_rm(const std::string& project_path, bool verbose) {
    // derives ai_user from project_path
    std::string ai_user = /* derived from path */;
    // ... proceeds to unmount, remove home, remove SSH key
    // MISSING: validate_ai_user_ownership(ai_user)
}
```

## Invariant Violated

**I1**: "User can only operate on own resources" — VIOLATED

## Fix Suggestion

Add `validate_ai_user_ownership(ai_user)` at the top of `cmd_rm()`, identical to the check in `cmd_mkdir`, `cmd_touch`, `cmd_cp`, `cmd_mv`, and `force_destroy`.

```cpp
int cmd_rm(const std::string& project_path, bool verbose) {
    // ... derive ai_user ...
    if (!validate_ai_user_ownership(ai_user)) {
        utils::get_logger()->error("rm: caller does not own ai-user '{}'", ai_user);
        return 1;
    }
    // ... rest of function
}
```
