# SEC-024: Symlink Chown Fallback Uses chown() Instead of lchown()

## Description
In `safe_chown_single()`, when `open(O_NOFOLLOW)` fails with `ELOOP` (indicating the path is a symlink), the code falls back to using `chown()` which **follows symlinks**, defeating the `O_NOFOLLOW` protection.

## Code Evidence
```cpp
// commands.cpp:132-152
static bool safe_chown_single(const fs::path& p, uid_t uid, gid_t gid) {
    int fd = open(p.c_str(), O_RDONLY | O_NOFOLLOW);  // O_NOFOLLOW prevents symlink open
    if (fd < 0) {
        if (errno == ELOOP && fs::is_directory(p)) {  // fs::is_directory FOLLOWS symlinks!
            if (chown(p.c_str(), uid, gid) != 0) {     // chown() FOLLOWS symlinks!
                ...
            }
            return true;
        }
        ...
    }
    ...
}
```

## Formal Analysis
### Invariant Violation: I4 (No symbolic link traversal attacks)
**Proof of Violation:**
1. Attacker creates symlink `attack_link -> /target_dir`
2. `safe_chown_single("attack_link", uid, gid)` is called
3. `open(O_NOFOLLOW)` fails with `ELOOP` (symlink detected)
4. `fs::is_directory("attack_link")` **follows the symlink**, sees `/target_dir` is a directory
5. Condition `errno == ELOOP && fs::is_directory(p)` is TRUE
6. `chown("attack_link", uid, gid)` **follows the symlink**
7. Ownership of `/target_dir` is changed (outside intended scope)

## Attack Scenario
1. Main user creates: `/home/main/safe_dir/my_link -> /etc/some_config`
2. Calls `am cp /home/main/safe_dir/file /home/main/safe_dir/my_link -u imain_test`
3. `safe_chown_path` recursively chowns `/home/main/safe_dir/my_link` (destination)
4. `safe_chown_single` on symlink `my_link`:
   - open fails with ELOOP
   - fs::is_directory sees `/etc/some_config` is a directory
   - chown changes ownership of `/etc/some_config`

## Suggested Fix
```cpp
if (errno == ELOOP) {
    // Use lchown() which does NOT follow symlinks
    if (lchown(p.c_str(), uid, gid) != 0) {
        utils::get_logger()->error("safe_chown: lchown({}) failed: {}", p.string(), strerror(errno));
        return false;
    }
    return true;
}
```

**Note:** Remove the `fs::is_directory(p)` check - it's unnecessary and dangerous.
