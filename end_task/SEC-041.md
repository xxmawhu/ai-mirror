# SEC-041: is_path_allowed TOCTOU with Canonical Resolution

## Severity
MEDIUM (CVSS 5.3)

## Category
TOCTOU / path_validation

## Component
shell.cpp:237-258 `is_path_allowed()`

## Description
The `is_path_allowed()` function uses `fs::canonical(p)` to resolve the path before checking against the user's home directory. However, `canonical()` requires the path to exist. If the path doesn't exist, the function returns false.

**Issue**: For new file creation operations (cmd_touch, cmd_cp to new destination), the path may not exist yet, blocking legitimate operations.

**TOCTOU**: More critically, between canonical resolution and actual operation, a symlink can be replaced, changing the resolved target.

## Formal Proof
```
Legitimate case blocked:
T1: User wants to create /home/user/newdir/newfile.txt
T2: PathResolver::resolve() fails (path doesn't exist)
T3: is_path_allowed() returns false
T4: Operation rejected (false negative)

TOCTOU case:
T1: User requests operation on /home/user/existing_link -> resolves to /home/user/safe_target
T2: is_path_allowed(/home/user/existing_link) returns true (canonical = /home/user/safe_target)
T3: [TOCTOU WINDOW] symlink replaced: ln -sf /etc /home/user/existing_link
T4: Actual operation now targets /etc
```

## Affected Code Paths
- `cmd_create()` line 65: checks resolved path
- `cmd_mkdir()` line 209: same
- `cmd_touch()` line 259: same
- `cmd_cp()` line 323-328: source and destination
- `cmd_mv()` line 382-388: same

## User Scenario Blocked
User wants to create a new project at `/home/user/newproject/` where the directory doesn't exist yet. `cmd_create()` will reject because canonical resolution fails.

## Recommendation
1. For new file/dir creation, use `weakly_canonical()` on parent directory
2. Verify parent is allowed, not the full path
3. Use O_NOFOLLOW when opening for creation
4. After creation, verify the actual location is under home

## Test Case
```bash
# Legitimate case blocked:
mkdir /home/user/newproj  # directory exists
touch /home/user/newproj/newfile.txt  # file doesn't exist yet
sudo ai-mirror touch /home/user/newproj/newfile.txt ai_user_proj
# Result: "Invalid path" (canonical fails on non-existent file)
```
