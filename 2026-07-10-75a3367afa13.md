# 持久内容: cmd_cd mount 检查误报分析与修复

## 问题

`am create` 后立即 `am cd` 误报：
```
warning: 3 mount(s) not configured for this project:
- /mnt/beegfs_data/usr/maxx/.share/
- /mnt/beegfs_data/usr/maxx/.local/bin/
- /mnt/beegfs_data/usr/maxx/.local/lib/
```

实际 mounts 已正确配置。

## 根因

`cmd_cd()` 和 `mount_watch` 使用**原始配置路径**（raw config path）计算预期 mount target，而 `do_configure()` 先通过 `PathResolver::resolve()` canonicalize 路径后再创建 bind mount。配置路径有尾部斜杠时导致比对失败。

## 数学证明

### 定义

| 符号 | 含义 |
|------|------|
| `c` | 配置中的 mount path |
| `R(c)` | `PathResolver::resolve(c)` — canonicalize，去尾部斜杠 |
| `T(p)` | `to_ai_user_path(p)` — 主用户路径→AI用户路径转换 |

### 引理

1. `fs::weakly_canonical()` 去除尾部斜杠 ✓
2. `/proc/mounts` 中 mount target 从不含尾部斜杠（实测 1986 entries 均无）✓
3. `fs::path::operator/` 保留 relative 部分的尾部斜杠 ✓

### 定理

若配置路径 `c` 以 `/` 结尾，则 `T(c) != T(R(c))`，导致 `status_targets.find(T(c)) == end()` → 假阳性

**证明**:
- `R(c)` 去尾部 `/` → `T(R(c))` 无尾部 `/`
- `T(c)` 保留尾部 `/` → `T(c)` 有尾部 `/`
- `.am_status` 存储 `T(R(c))`（来自 `/proc/mounts`）
- `T(c) != T(R(c))` → 找不到 → 误报

## 修复

在 `cmd_cd()` 和 `mount_watch` 的 mount 检查中，先 `PathResolver::resolve(mp)` 再 `to_ai_user_path()`，与 `do_configure()` 行为一致。

- `src/cli/commands.cpp` — `cmd_cd()` mount 检查
- `src/tools/mount_watch.cpp` — 相同逻辑
