# SEC-040: Recursive Chown TOCTOU via Symlink Injection

## Severity
HIGH (CVSS 7.5)

## Category
symlink_traversal / TOCTOU

## Component
commands.cpp:154-182 `safe_chown_path()`

## Description
The `safe_chown_path()` function recursively iterates over a directory and calls `safe_chown_single()` for each file. After the parent directory's ownership is changed to ai_user (line 163), the ai_user gains write access to that directory.

**TOCTOU vulnerability**: During the recursive iteration:
1. Parent directory ownership is changed first
2. ai_user can now create new entries (symlinks) in the directory
3. The iterator discovers newly created symlinks
4. `safe_chown_single()` triggers ELOOP for symlinks
5. `lchown()` operates on symlink target (see SEC-024)

## Formal Proof
```
States: S0 (parent dir chowned) -> S1 (ai_user has write) -> S2 (symlink created) -> S3 (iterator finds symlink) -> S4 (lchown on target)

Attack trace:
T1: safe_chown_path("/home/user/project", "ai_user_proj")
T2: safe_chown_single(parent_dir, ai_uid, ai_gid) succeeds
T3: [TOCTOU WINDOW] ai_user creates symlink: ln -s /root/.ssh/id_rsa /home/user/project/escape
T4: fs::recursive_directory_iterator discovers new symlink
T5: safe_chown_single("/home/user/project/escape", ai_uid, ai_gid)
T6: open() fails with ELOOP
T7: lchown("/home/user/project/escape") → /root/.ssh/id_rsa ownership changed
```

## Root Cause
The security model violates the principle: "ownership should not be granted before all contents are verified safe"

## Affected Code Paths
- `cmd_cp()` → `safe_chown_path()` (line 340)
- `cmd_mv()` → `safe_chown_path()` (line 401, 419)

## Impact
An ai_user can inject symlinks during the ownership transfer process and escalate to modify system file ownership.

## Recommendation
1. **Two-phase approach**: First collect all entries, validate none are dangerous symlinks, then change ownership in reverse order (children first, parent last)
2. **Use file descriptor approach**: Open parent with O_PATH, use fchownat() with AT_SYMLINK_NOFOLLOW for children
3. **Block symlink creation**: Set parent directory permissions to block symlink creation during transfer
4. **Verify ownership change**: After chown, verify no new entries appeared

## Test Case
```bash
# Setup with concurrent attack:
sudo ai-mirror cp /home/user/src /home/user/dst --user ai_user_proj
# During copy (as ai_user if timing allows):
ln -s /etc/shadow /home/user/dst/src/new_symlink
# Result: /etc/shadow ownership changed to ai_user_proj
```
