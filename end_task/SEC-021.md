## Description

`cmd_touch()` has a TOCTOU (Time-of-Check-Time-of-Use) race condition. It first checks `!fs::exists(file_path)` to decide whether to create the file, then calls `recursive_chown(file_path, ai_user)`. Between the exists check and the chown, an attacker can replace `file_path` with a symlink pointing to a sensitive system file (e.g., `/etc/shadow`, `/etc/passwd`), causing `chown` to change ownership of the target.

## Attack Scenario

1. Attacker has write access to the parent directory (e.g., their project dir)
2. Attacker runs `am touch /home/attacker/project/newfile` in one terminal
3. Concurrently, attacker replaces `/home/attacker/project/newfile` with a symlink to `/etc/shadow`
4. `recursive_chown` follows the symlink and changes `/etc/shadow` ownership to ai_user

**Exploitation difficulty**: Medium — requires precise timing but can be automated with inotify.

## Affected Code

```cpp
// src/cli/commands.cpp:195-215
int cmd_touch(const std::string& path, const std::string& ai_user, bool verbose) {
    // ...
    if (!fs::exists(file_path)) {
        // create the file
    }
    // RACE WINDOW: attacker replaces file_path with symlink
    if (!recursive_chown(file_path, ai_user)) {
        // ...
    }
}
```

## Fix Suggestion

Use `O_NOFOLLOW` via `fchownat()` with `AT_SYMLINK_NOFOLLOW` flag, or verify the file after creation using file descriptor-based operations:

```cpp
// After creating the file, open it with O_NOFOLLOW to get an fd
int fd = open(file_path.c_str(), O_RDONLY | O_NOFOLLOW);
if (fd < 0) {
    // Symlink detected — abort
    return 1;
}
if (fchown(fd, pwd->pw_uid, pwd->pw_gid) != 0) {
    close(fd);
    return 1;
}
close(fd);
```

Alternatively, verify inode consistency between pre-check and post-check.
